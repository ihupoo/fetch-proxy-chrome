Mock 语法说明

适用于 HTTP 规则中的 MockJS-like Template 模式。

使用步骤

1. 将“响应生成方式”切到 MockJS-like Template。

2. 在 responseBody 中填写合法 JSON 模板。

3. 规则在 quick / full 模式下都会按模板动态生成响应。

属性规则

name|18-40：生成整数区间值。

score|1-100.1-2：生成浮点数，整数部分范围 1-100，小数位 1-2 位。

id|+1：按步长递增。

list|2-4：数组生成 2 到 4 项。

type|1：从数组中随机选 1 项返回。

占位符

支持：@guid @uuid @cname @name @integer(1,100) @float(1,100,1,2)

也支持：@pick('open','closed') @word @sentence @paragraph @date @time @datetime

网络相关：@email @url @domain @ip @color

示例 1：用户列表

{
  "code": 0,
  "message": "ok",
  "data|2-4": [
    {
      "id|+1": 1000,
      "name": "@cname",
      "age|18-40": 1,
      "email": "@email",
      "status": "@pick('enabled','disabled')"
    }
  ]
}

示例 2：分页结果

{
  "code": 0,
  "page": 1,
  "pageSize": 10,
  "total|80-120": 1,
  "list|5-10": [
    {
      "orderId": "@guid",
      "title": "@sentence",
      "amount|50-500.1-2": 1,
      "createdAt": "@datetime"
    }
  ]
}

注意

模板内容必须是合法 JSON，不能直接写 JS 表达式或函数。

当前不是完整 mockjs 官方实现，而是兼容常用语法的轻量版本。

Replace 规则说明

当“操作方式”选择 Replace (替换响应) 时，规则会基于原始响应内容做文本替换。

生效方式

系统会先拿到接口原始响应体，再把其中所有出现的 replaceFrom 文本替换成 replaceTo。

当前是普通字符串替换，不是正则替换，也不会解析 mock 模板。

如果没有填写 replaceFrom，但填写了响应体，那么会直接用响应体整体覆盖原始响应。

示例 1：替换 JSON 里的字段值

原始响应:
{"status":"offline","name":"demo-user"}

replaceFrom:
offline

replaceTo:
online

结果:
{"status":"online","name":"demo-user"}

示例 2：替换整段文本

原始响应:
hello demo world

replaceFrom:
demo

replaceTo:
ajax-proxy

结果:
hello ajax-proxy world

注意

这是纯文本替换，所以 JSON、HTML、JS、普通文本都可以用，但必须保证替换后的内容仍然是你期望的格式。

WebSocket 规则说明

WS 规则按消息方向做纯文本替换，不区分消息是不是 JSON。

生效方式

Outgoing 表示客户端发出的消息。命中后，会在消息真正发出前，把其中的 outgoingFind 替换成 outgoingReplace。

Incoming 表示服务端返回给页面的消息。命中后，会在页面收到消息前，把其中的 incomingFind 替换成 incomingReplace。

当前是普通字符串替换，不是正则，也不会解析 mock 模板。

示例 1：改写客户端发出的订阅参数

客户端原始发送:
{"action":"subscribe","room":"room-a"}

outgoingFind:
room-a

outgoingReplace:
room-b

实际发出:
{"action":"subscribe","room":"room-b"}

示例 2：改写服务端返回状态

服务端原始返回:
{"type":"status","value":"offline"}

incomingFind:
offline

incomingReplace:
online

页面最终收到:
{"type":"status","value":"online"}

注意

如果查找文本为空，对应方向就不会做替换。

替换发生在文本层，所以请确保替换后仍然符合你的业务协议格式。
